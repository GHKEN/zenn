# react-hook-form useFieldArray id 問題デモ

react-hook-form の `useFieldArray` で `id` プロパティが上書きされる問題と、その解決方法を実際に確認できるデモアプリケーションです。

## 問題の概要

`useFieldArray` は、内部で一意の `id` を各フィールドに自動生成します。このため、元のデータに `id` フィールドが含まれていると、それが上書きされてしまいます。

## デモの内容

このアプリケーションには3つのタブがあり、それぞれ異なるアプローチを示しています：

1. **問題の再現**: `id` が上書きされる問題を実際に確認できます
2. **解決策1: keyName**: `keyName` オプションを使用する方法
3. **解決策2: userId（推奨）**: フィールド名を `userId` に変更する方法

## セットアップ

```bash
# 依存関係のインストール
bun install

# 開発サーバーの起動
bun run dev
```

ブラウザで http://localhost:5173 を開いてください。

## 使い方

1. 各タブを切り替えて、異なるアプローチを確認
2. 「問題の再現」タブで元のIDが上書きされることを確認
3. 「解決策1: keyName」では`keyName`オプションで元のIDを保持
4. 「解決策2: userId」ではフィールド名変更で問題を回避
5. フォームに入力して「ユーザーを追加」「削除」を試してみる
6. 「送信」ボタンをクリックして、コンソールで送信データを確認
